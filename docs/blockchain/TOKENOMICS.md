# Tokenomics

Mathematical Authority: See ../../MATH.md.
Optionality Rule: Blockchain is enabled only when BC > 1.

Blockchain is an optional settlement subsystem, not a core DCCO dependency.
It is justified only when the blockchain score BC > 1 as defined in MATH.md.
Otherwise, DCCO must use conventional databases, event stores, and audit logs.

## DCC Token

`DCC` is a utility settlement token, not a speculative asset.

`DCC` exists only for settlement, reservation, carbon accounting, inter-site exchange, and smart contract execution where the blockchain score `BC > 1`.

## Asset Classes

- `DCC`: settlement token
- `CCC`: carbon credit certificate
- `RCC`: renewable capacity certificate
- `CPU`: compute unit
- `GPU`: accelerator unit
- `NCU`: network capacity unit
- `SCU`: storage capacity unit
- `DRU`: disaster recovery unit
- `ECU`: energy capacity unit
- `CCU`: cooling capacity unit
- `SLA`: service-level credit unit

## Constraint

Token design must improve economic coherence `Xi` without increasing cost, obstruction, or noise enough to reduce DCCO.

Token issuance must be backed by measured or contractually reserved capacity, verified environmental instruments, or approved settlement obligations. Unbacked speculative issuance is outside the DCCO model.
